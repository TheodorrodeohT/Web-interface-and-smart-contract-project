# Павлов Федор Игоревич, 166-2, Разработка веб интерфейсов для криптовалют и смарт-контрактов

### Описание актуальности решаемой задачи, обзор существующих решений
Умные контракты, лежащие в основе blockchain логики многих продуктов, - по сути своей абстрактные объекты. Каждый из них обладает уникальным адресом, на который можно отправлять транзакции (аналогия с api вызовами). Стандартный интерфейс для них - криптовалютный кошелек предоставляет минималистичный интерфейс где можно задать адрес, вызываемый метод и payload (двоичный код с данными). Для комфортного использования умных контрактов предусмотрена возможность расширять пользовательский интерфейс с помощью стандартных технологий front-end разработки (HTML + CSS + Javascript). В рамках этого проекта будет необходимо реализовать пользовательский интерфейс к одному из смартконтрактов.

На сегодняшний день ситуация непосредственно с самими [смарт-контрактами](https://www.stateofthedapps.com/) неординарная. Действительно хороших проектов, в силу ограниченного функционала самого принципа смарт-контракта, катастрофически мало: одним из единственных наиболее успешных продуктов за последнее время можно привести пример [CryptoKitties](https://www.cryptokitties.co/), использующих Ethereum смарт-контракт.

Именно поэтому этот вопрос стоит наиболее актуально, так как помимо front-end для готового и функционирующего смарт-контракта существует возможность придумать собственную уникальную идею контракта и реализовать её.

### Обзор используемых технологических решений
Для работы с front-end будут использоваться такие языки разметки, как HTML и CSS, плюс язык JavaScript.

Технологической основой являются [блокчейн технологии](https://ru.wikipedia.org/wiki/Блокчейн), так как исследуемая криптовалюта (Bitcoin, Ethereum и пр.) с ними непосредственно связаны.

### План работы реализации функциональности проекта
1. Изучение рынка смарт-контрактов и выбор (или создание) лучшего кандидата (конец 2017 - начало 2018 года);

2. Создание прототипа с минимальным функционалом (к середине марта). Начало работы над сопроводительной документацией;

3. Доработка функционала приложения, доработка документации (апрель - начало мая);

4. Доработка дизайна приложения (конец мая);

5. Представление проекта на мероприятии или интеграция с внешним заказчиком (июнь - август).

## Смарт-контракт: [smartcard.io](https://github.com/TheodorrodeohT/Web-interface-and-smart-contract-project/tree/master/smartcardio)

Для КТ-3 [я](https://github.com/theodorrodeoht), [Чистяков Глеб](https://github.com/glebelg) и [Цхай Борис](https://github.com/TheLons) реализовали смарт-контракт и веб-интерфейс для него. Разработкой занимались все трое, в то время как в мои обязанности входило формирование веб-интерфейса на поздних стадиях проекта.

**Проект реализован** на блокчейн-платформе Ethereum. Были использованы следующие языки программирования и разметки: Solidity, HTML, CSS, Javascript, а также [JavaScript API: web3js](https://github.com/ethereum/wiki/wiki/JavaScript-API). 

![фронт пейдж](https://i.imgur.com/ZTG6TiI.png)

**Суть смарт-контракта**: пользователь заполняет данные о себе в специальной форме (на текущий момент это имя, фамилия, возраст), которую можно перенастроить на запрос любых других полей.

![секонд пейдж](https://i.imgur.com/TRs9okO.png)

**После этого** открывается окно подтверждения транзации (в нашем случае [MetaMask](https://metamask.io/)), где утверждается её стоимость (gas). После успеха исчезает значок загрузки и внизу появляется принятая контрактом строка.

![metamask](https://i.imgur.com/82zBXl2.png)

**В конце** имеем список всех добавленных визитных карт, в котором каждый пользователь имеет возможность повысить рейтинг любой из карт. Каждая визитка, в том числе, имеет статус: confirmed, unconfirmemd, выставляемый модератором сайта.

![база](https://i.imgur.com/zBBKbpX.png)
